   Purpose: Show the working executable in action

   Purpose: Prove it works (not just theory)

================================================================================
RESPONSIVE BREAKPOINTS
================================================================================

Desktop (> 1024px)
- Side-by-side layout for hero (diagram + code)
- Full-width ASCII art (monospace readable)
- Large training curve (800px width)

Tablet (768px - 1024px)
- Stacked layout for hero (diagram above code)
- Scrollable ASCII art container
- Scaled training curve (100% width)

Mobile (< 768px)
- Single column layout
- Smaller ASCII art font (8px)
- Compressed training curve
- Readable code snippet (smaller font)

- Line height: 1.2
- Background: dark (#1e1e1e)
- Foreground: green (#00ff00) or white (#ffffff)

ASSET LOADING STRATEGY
================================================================================

Critical (inline for speed):
- architecture.svg (6.6 KB) - INLINE in HTML
- training-curve.svg (7.0 KB) - INLINE in HTML
- Above-the-fold CSS - INLINE in <head>

Deferred (can load async):
- mnist-ascii-clean.txt (6.3 KB) - Can be lazy-loaded
- Below-the-fold CSS - Separate file

Don't load:
- mnist-ascii.txt (has build warnings, use clean version)

Total critical path: ~15 KB (very fast!)
